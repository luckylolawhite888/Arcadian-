### 4. 💡 THE 5-SECOND DELAY MYSTERY (Critical Fix)
**Problem:** Every page request took exactly 5 seconds via HTTPS (0.008s via HTTP)
**Root Cause:** Ghost's image size probe (`image-size.js`) was timing out trying to probe an external URL
**Found in database:**
- `cover_image` setting was `https://static.ghost.org/v5.0.0/images/publication-cover.jpg`
- Ghost's `_probeImageSizeFromUrl()` uses `probe-image-size` with a 5s timeout
- On every page request, Ghost tried to probe this external image and waited 5s for timeout
- The probe failure was due to Tor proxy being unhealthy (Ghost tried to route through it)
**Fix:** `UPDATE settings SET value = "" WHERE key = "cover_image";`
**Result:** Response time dropped from 5.0s → 0.025s

### 5. Host Nginx Activation (The Real Fix for thenewworldorder.io)
**Problem:** Maya's browser couldn't reach thenewworldorder.io (ERR_CONNECTION_TIMED_OUT)
**Root Cause:** Docker containers (`tnwo-ghost-proxy`) were occupying ports 80/443, blocking **iRedMail's host nginx** which was the original web server
**Revealed in logs:**
- iRedMail was installed with three nginx configs:
  - `00-main-site-443.conf` → port 443 SSL for thenewworldorder.io
  - `00-main-site.conf` → port 444 (fallback, conflicted)
  - `00-mail-ssl.conf` → mail subdomain
- Host nginx was `inactive (dead)` because it couldn't bind to 80/443 (Docker had them)
**Fix:**
1. `docker stop tnwo-ghost-proxy` and `docker stop tnwo-network-nginx-proxy-manager-1`
2. `systemctl start nginx` — host nginx started serving on 80/443/444
3. `systemctl enable nginx` — ensures it starts on boot
4. Disabled conflicting port 444 config: `mv 00-main-site.conf 00-main-site.conf.bak`
**Lesson:** ALWAYS check host services before creating Docker containers on the same ports

### 6. Cloudflare Reconnection
- Purged cache: `purge_cache` API with `purge_everything: true`
- Toggled DNS record between proxied/DNS-only multiple times
- **After host nginx fix** — Cloudflare proxy started working immediately (0.07s response via proxy)
- **SSL setting:** Changed from `strict` to `full` (accepts any valid cert on origin)

## 💡 LESSONS LEARNED
1. **Ollama is a RAM hog** — needs memory limits or dedicated server
2. **Host nginx > Docker nginx** for production web serving with iRedMail
3. **Ghost image probe** can cause 5s delays if external images are unreachable
4. **SSH key must be stored in a persistent volume** or it's lost on container rebuild
5. **Docker bridge DNS fails under memory pressure** — use host networking for critical services
6. **Cloudflare 522** is usually a firewall/port conflict issue, not an actual Cloudflare problem
7. **Always check Docker containers don't conflict with host services** on same ports
