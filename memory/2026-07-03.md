## Chat FIXED ✅ — Root Cause: ACCESS_CODE Never Set After Login (01:00 UTC)
**The REAL root cause** — not the async issue I kept chasing:
- `attemptLogin()` fetches auth, gets `{"valid":true}`, calls `unlock()`
- **`unlock()` never stored the access code** into `ACCESS_CODE` variable!
- `apiPost` checks `if(ACCESS_CODE) hdrs['x-access-code']=ACCESS_CODE`
- `ACCESS_CODE` was always empty → no auth header → server 401 → FALLBACK_REPLIES
- **Fix:** Added `ACCESS_CODE=code;` in the login success path: `if(res ? res.valid : code === DEMO_CODE){ ACCESS_CODE=code; unlock(); }`
- `sendMessage` was already `async function`, apiPost was already passing headers — they were working correctly. The missing piece was the actual code value.
