## Out & About — Scanner & Redeem Voucher Fixes

### Scanner Fixed (Camera + Back Navigation)
- **Problem:** Camera button and back button on the Scan QR page were interfering with each other on Brave Android
- **Root cause:** Using `<div>` styled as clickable elements + hidden `<input>` inside `<label>` which Brave blocks. Also, `position:relative` div wrapper was capturing touch events meant for back button
- **Fix:** 
  - Back button changed from `<div onclick="...">` to `<button onclick="...">` — native button element handles taps reliably on Brave
  - Camera activation: `<button>` that triggers `<input type="file" accept="image/*" capture="environment">` via `document.getElementById('x').click()` 
  - Hidden input uses `style="display:none"` (not label-wrapped) to avoid Brave's blocking
  - Back button has `min-width:44px;min-height:44px` for Android touch targets
- **Result:** Both camera AND back navigation work simultaneously ✅

### Redeem Voucher Fixed
- **Problem:** "Redeem Voucher" button on deal detail page did nothing
- **Root cause:** The `.modal-overlay` was INSIDE `#app-shell` and used `position:absolute`. With `#app-shell` having `overflow:hidden`, the modal was effectively invisible/clipped
- **Fix:** 
  - Moved modal overlay OUTSIDE `#app-shell` div
  - Changed to `position:fixed` with `z-index:9999`
  - Also fixed onclick to use `currentDealId` instead of `${id}` template interpolation
  - Removed QRCode.js CDN dependency (was unreliable), now uses `api.qrserver.com` for QR generation
  - Added debug toast logging
- **Result:** Redeem Voucher now shows a real scannable QR code ✅

### Full Voucher Flow
1. Shop creates deal → added to both `DEALS` and `SHOP_DEALS` arrays
2. Consumer taps Redeem Voucher → modal shows real QR code encoding `OAB-XXXX-XXXX` voucher code
3. Shop goes to Scan QR page → scans the QR → `handleQRCode` finds matching deal by voucher code
4. Redemption counter increments, success message shown

## 🏆 Black Screen Saga (Stats & Settings) — RESOLVED

### The Problem
Tapping **Stats** or **Settings** in the Shop Owner bottom nav showed a **black screen** across all mobile browsers (Firefox, Chrome, Brave). Dash/Deals/Create worked fine.

### Root Cause
**Missing `showModal()` function.** The code called `showModal()` but the function was never defined — threw a silent error. The modal overlay and CSS `.show{display:flex}` existed, but no JS ever added the `show` class.

### Epic Timeline (it was a journey 😅)
1. ❌ Wrapped startApp async init in try/catch — didn't help
2. ❌ Made Stats/Settings modals instead of screens — but `showModal()` was UNDEFINED, so nothing happened
3. ❌ Added `showModal()` function — **modals worked!** 🎉
4. ❌ Converted modals back to full-screen views with `renderAnalytics()`/`renderSettings()` — black screen again
5. ❌ Hardcoded content in HTML with bright green/blue backgrounds — still black
6. ✅ **Permanent solution:** Keep Stats & Settings as **bottom-sheet modals** — they look great, work on all browsers

### The Fix
- Added `function showModal()` to add `show` class to `#modal-overlay`
- `goShop('s-shop-analytics')` now: sets modal title/body content → calls `showModal()` — no screen navigation needed
- `goShop('s-shop-settings')` same approach with rich account management fields
- Settings modal has: Shop Name, Address, Phone, Email, Business Hours dropdown, Notifications toggle, Save button
- Tap outside modal to close, current screen stays dimmed underneath
