## Phase 1 Bug Fix ‚Äî "Other" & Create Deal Black Screen (12:27 UTC)
- **Root Cause:** `otherCat` state variable (`oc`, `soc`) was supposed to be inserted after the CATS array declaration but search pattern `"Other"],` (comma) didn't match actual code `"Other"];` (semicolon). So `soc` was undefined ‚Üí ReferenceError on "Other" click ‚Üí React crashed entirely, including create deal.
- **Fix:** Replaced `"Bar","Other"];` with `"Bar","Other"];\n  var otherCat = u(""), oc = otherCat[0], soc = otherCat[1];`
- **Lesson:** Always check exact source code patterns when doing text replacements ‚Äî backup files can have subtle differences (comma vs semicolon).
- **Verification:** JS syntax checks passed, all variables found in source.

## Phase 1 Bug Fix #2 ‚Äî Create Deal Black Screen (12:32 UTC)
- **Root Cause:** Previous build renamed ALL component props (`profile`‚Üí`pr`, `purchases`‚Üí`pu`, etc.) to save space. This broke the create deal page because some child component still referenced the original prop names internally.
- **Fix:** Rebuilt from clean backup keeping ALL original prop names (`profile`, `purchases`, `setProfile`, `setPurchases`). Only added `l` and `sl` to the props object. No prop renames.
- **Also fixed "Other" category:** The `otherCat` state (`oc`, `soc`) was actually inserted but the create deal crash prevented testing it.
- **Lesson:** Prop renaming is fragile ‚Äî only do it when absolutely necessary and verify every single reference. When a black screen appears, it's almost always an undefined variable or a syntax error deep in the render tree.

### Backend Integration Attempt (FAILED)
- Built a Python script to inject Supabase client + API calls into Maya's v3 HTML
- Deployed 580KB merged file to the server
- **Result: App broke ‚Äî buttons didn't respond** ("Browse Deals", "I'm a Shop Owner", "Browse without account" all unclickable)
- **Root Cause Analysis:**
  - Syntax check passed (braces=0, parens=0, brackets=0)
  - `new Function()` in Node failed on `async function` (not a real error ‚Äî browsers handle it fine)
  - Functions `showLogin`, `goConsumer`, `goScreen` all existed in the merged script
  - Backtick/template literal escaping caused issues in the injection
  - Restored backup ‚Äî turns out `index_backup_may13.html` was the **React version (26KB)**, NOT Maya's vanilla JS build
- **Current state:** Maya's original v3 restored at `/oatest/` (her 568KB file) ‚Äî **waiting for her to test if buttons work on the clean original**

### Key Lesson
- Backend injection into a large single-file app is fragile. Better approach: add backend as a **separate `<script>` tag** after the main script instead of injecting inline.
- The backup/rollback strategy needs care ‚Äî make sure the right backup is being used.
- Always verify the app works BEFORE deploying backend changes.
