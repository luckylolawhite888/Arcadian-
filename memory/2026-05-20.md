### Phase 5: TLS Fingerprint Mismatch (13:56+)
- Maya scanned QR code (v4, manually encoded) — app showed "TLS fingerprint mismatch"
- Decoded QR payload: `{"url":"wss://jenny.thenewworldorder.io","bootstrapToken":"NnrIkBnoRibWKxaeyGS5P9xXFC6jOcb-1KNvbOJsJw8z8"}`
- No fingerprint field embedded in setup code (as expected for `--remote` or manually encoded QRs)
- Verified nginx TLS: `openssl s_client -connect 127.0.0.1:443` shows full chain, hostname verified
- **Root cause theory**: The Android app validates the TLS certificate fingerprint against the QR setup code. With manual/remote QRs having no embedded fingerprint, the app uses its own CA trust store to verify. If the app's trust store doesn't have the Let's Encrypt root (unlikely but possible on some Android builds), or if the cert chain isn't being sent correctly, it would fail.
- **Resolution path**: Suggested Maya try **manual connect** (enter URL + bootstrap token by hand) instead of QR scanning, which might bypass whatever fingerprint validation chain is failing.
- **Lesson**: TLS fingerprint matching between QR and app is fragile. Manual entry with known-good certificate chain may work where QR doesn't.
