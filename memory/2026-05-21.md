## 🐝 Jenny's Crash Loop & Fix (01:30–02:00)
- **The problem:** When Lola ran `npm install -g openclaw@latest`, it overwrote `/usr/lib/node_modules/openclaw/` — Jenny's old v2026.4.2 gateway (PID 33250) was still running from these files, but the kernel module filenames changed between versions → "Cannot find module" crash
- **First fix (01:40):** `kill -TERM 33250` → Jenny respawned on PID 57775 but config was broken
- **Second attempt (01:50):** Jenny auto-restarted to PID 61024 after the 33750 was killed — still broken
- **Root cause discovered (01:54):** Jenny's config had:
  1. `Telegram enabled: None` — missing the `enabled: true` field
  2. `Groups config: {}` — group config was lost
  3. `hooks.token` matched gateway auth token — v2026.5.19 rejects this
  4. `streaming: "partial"` must be an object `{mode: "partial"}` in v2026.5.19
- **Final fix (01:55):** Rewrote Jenny's config with proper Telegram enabled, group config, distinct hooks token, gateway auth token, and streaming as object
- **Jenny came alive:** PID 62508, port 19200, logs show `"Inbound message telegram:1523950034 -> @jennytheworkerbot"` and `"Inbound message telegram:group:-1003778508955 -> @jennytheworkerbot"` — she's receiving DMs AND group messages!
- **Lesson:** When two OpenClaw instances share the same `/usr/lib/node_modules/openclaw/`, upgrading one breaks the other. Config fixes needed for v2026.5.19: streaming as object, distinct hooks token, explicit telegram.enabled

## 🌐 The Hive Chat Debugging — Day Session (12:00–13:00 UTC)

### The Bug: Messages vanish on send
- Maya can log into The Hive chat, type a message, press enter → text disappears from input but doesn't appear in chat area
- BUT her messages **are** stored in the SQLite database successfully (IDs 5-10)
- The Socket.IO broadcast (`io.emit('message', ...)`) wasn't reaching her browser

### Root Cause Found by Jenny 🐝
- **Global CSP header** in `/etc/nginx/conf-enabled/headers.conf`:
  `default-src https: data: 'unsafe-inline' 'unsafe-eval'`
- `default-src` covers `connect-src` → only allows `https:` and `data:` protocols
- This blocks WebSocket connections (`wss://`) in the browser
- Messages saved via HTTP fetch (not blocked) but live broadcast via Socket.IO WebSocket was nuked

### Fix Applied (by Jenny via SSH to IONOS host)
- Updated CSP to: `default-src https: data: 'unsafe-inline' 'unsafe-eval'; connect-src https: wss: ws:`
- Reloaded nginx
- This allows WebSocket connections through

### Additional Fixes (by Lola)
1. **Double-path bug fix** — Changed `io("/hive", {path: "/hive/socket.io"})` to `io({path: "/hive/socket.io"})` — the first arg was adding an extra `/hive` prefix, making Socket.IO try to connect to `/hive/hive/socket.io/...`
2. **HTTP polling fallback** — Rewrote `app.js` to send messages via `POST /hive/msg` HTTP endpoint instead of `socket.emit('message', ...)`, and added `setInterval` polling every 2s to refresh messages
3. **Added `/hive/msg` POST endpoint** to server.js — accepts `{text, username}`, inserts to SQLite, broadcasts via `io.emit`
4. **Cache-busting** — Added `?v=3` to CSS/JS script tags
