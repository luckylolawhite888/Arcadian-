## 2026-06-22 — Free APIs Wired into Out & About Server; Briefing Discussion

### 🏪 Out & About — Free API Endpoints Deployed
After significant effort (indentation bugs, wrong HTTP method routing), got 7 free API endpoints deployed into the Out & About server's `do_GET` method:

**✅ Live & Working (no auth keys):**
- `/btc` — Bitcoin price in USD/GBP/EUR (CoinGecko)
- `/quote` — Random quote (ZenQuotes)
- `/geocode?q=address` — Nominatim geocoding → lat/lng
- `/uk-holidays` — UK bank holidays (gov.uk)
- `/qr?text=URL` — QR code redirect (qrserver.com)

**❌ Blocked (need API keys):**
- `/nearby-stops` — TfL API now requires key (403)
- `/space-news` — Spaceflight News API 403'd

**Key technical lessons:**
- Server uses **8-space** indentation (not 4!)
- GET routes go in `do_GET`, not `do_POST`
- `json_response(self, data)` is a top-level function, not a method
- SFTP upload via `ssh2.Client.sftp()` → `createWriteStream()` works reliably for file transfers
- Base64 encoding in shell commands gets messy with single quotes

### 🐛 CesiumJS Globe Rendering Bugs

**Bug 1: 404 on globe app button**
- Grid app "Globe" pointed to `<a href="/globe">` but no route existed
- **Fix 1:** Added `/globe` route in FastAPI `main.py` serving `static/index.html`
- **Fix 2:** Updated HTML `<a href="/globe">` to point to `/globe` not `#`

**Bug 2: `setDynamicLighting is not a function`**
- Caused by `Cesium.Terrain.fromWorldTerrain()` creating terrain with missing API
- Happened on both CDN versions (1.119, 1.116) and both CDN sources (jsdelivr, unpkg)
- **Fix attempts:**
  - 🔴 Prototype patch `Cesium.Globe.prototype.setDynamicLighting = function(){}`
  - 🔴 Wrapped Viewer init in try/catch
  - 🔴 Switched to `EllipsoidTerrainProvider` → caused `Cannot read 'update' of undefined`
  - ✅ **Final fix:** Switched to `new Cesium.Viewer()` with **no terrain at all** — `scene.terrainProvider = new Cesium.EllipsoidTerrainProvider();` set **before** first animation frame
  - ✅ Downgraded to Cesium **1.120** (latest stable from unpkg)

**Bug 3: Cloudflare cache**
- Maya suggested Cloudflare was caching old HTML
- Purged cache via Cloudflare API: `https://api.cloudflare.com/client/v4/zones/e8f2ea4b9634485eab2c042416fec384/purge_cache`
- Used API key from `.vault/cloudflare.json`
- **Result:** Cache purged successfully (success: true)

**Bug 4: globe button click does nothing**
- The 🌍 button in the grid was a static `<a>` tag, but Maya expected it to toggle the Cesium globe
- **Fix:** Changed grid app button to call `toggleGlobe()` directly
